All done. Here's what was changed in `import-steam-prices.js`:

1. **`savedGameQueue` global** — captures the full ordered game ID list right before batching starts
2. **`saveState()`** — now persists `gameQueue` alongside `processedGameIds`
3. **Resume path** — when `savedState.gameQueue` exists, rebuilds the game list in the exact saved order (skipping processed, appending any newly-added games at the end). Falls back to old filter-only behavior for state files without a queue.
4. **Deterministic sort** — `getGamesWithSteamIds()` now sorts by `game_id` for non-staleFirst runs, so even without a saved queue the order is reproducible across runs.
