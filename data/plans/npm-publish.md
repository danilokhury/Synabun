## Plan: Publish SynaBun on NPM as a Single Global Package

### Goal
`npm install -g synabun` → `synabun` starts the full system (MCP server + Neural Interface + hooks + skills + onboarding).

### User Choices
- **MCP Build**: Build on install (postinstall runs `tsc`)
- **Install UX**: `npm install -g synabun`
- **Native deps**: Postinstall rebuild (keep rebuild-pty.js)

---

### Changes

#### 1. Update `package.json` (root)

Add/modify these fields:

```json
{
  "bin": {
    "synabun": "./setup.js"
  },
  "files": [
    "setup.js",
    "postinstall.js",
    "mcp-server/package.json",
    "mcp-server/tsconfig.json",
    "mcp-server/src/",
    "mcp-server/run.mjs",
    "mcp-server/data/.gitkeep",
    "neural-interface/package.json",
    "neural-interface/server.js",
    "neural-interface/public/",
    "neural-interface/templates/",
    "neural-interface/scripts/",
    "hooks/",
    "skills/",
    "data/.gitkeep",
    "CLAUDE.md",
    "README.md",
    "CHANGELOG.md"
  ],
  "scripts": {
    "start": "node setup.js",
    "postinstall": "node postinstall.js"
  },
  "keywords": [
    "claude",
    "memory",
    "mcp",
    "vector",
    "embeddings",
    "ai",
    "persistent-memory",
    "claude-code"
  ],
  "homepage": "https://synabun.ai"
}
```

**Why `files` whitelist**: Only ships what we list. Excludes node_modules (npm default), .env, data/*.json, data/*.log, mcp-server/dist/ (built on install), .DS_Store, .claude/, etc.

**Why no `mcp-server/dist/`**: User chose "build on install" — TypeScript compiles during postinstall.

#### 2. Create `postinstall.js` (new file)

Separate from setup.js because postinstall must NOT start the server. Handles:

1. Install neural-interface dependencies (`npm install` in neural-interface/)
2. Install mcp-server dependencies (`npm install` in mcp-server/)
3. Rebuild node-pty (run neural-interface/scripts/rebuild-pty.js)
4. Build MCP server TypeScript (`npm run build` in mcp-server/)
5. Create data directories if missing (data/, mcp-server/data/)

Reuses the same install/build check logic as setup.js (needsInstall, needsBuild) but without server launch, browser open, or banner.

#### 3. No changes to `setup.js`

setup.js already has `#!/usr/bin/env node` shebang and idempotent install/build checks (needsInstall, needsBuild skip if already done). When run as the `synabun` CLI command after postinstall, it detects everything is ready and goes straight to server launch.

### What ships in the NPM package (~6MB unpacked)

| Directory | Size | Purpose |
|-----------|------|---------|
| neural-interface/public/ | 4.6MB | Frontend UI |
| neural-interface/server.js | 600K | Express server |
| mcp-server/src/ | 560K | MCP TypeScript source |
| skills/ | 256K | Skill definitions |
| hooks/ | 176K | Claude Code hooks |
| neural-interface/templates/ | 12K | CLAUDE.md template |
| setup.js, postinstall.js | ~15K | CLI + install scripts |

### What does NOT ship
- `node_modules/` (npm excludes by default)
- `mcp-server/dist/` (built on install)
- `.env` (user-specific, generated by onboarding)
- `data/*.json`, `data/*.log` (runtime)
- `.claude/` (user-specific)
- `.DS_Store`, `Thumbs.db`
- `mcp-server/data/memory.db*` (runtime)
- `docs/` (dev docs, not needed for users)
- `tests/` (dev only)

### Install flow

```
npm install -g synabun
  └─ npm downloads package (~6MB)
  └─ postinstall.js runs:
     ├─ npm install in neural-interface/   (express, node-pty, playwright, ws, etc.)
     ├─ npm install in mcp-server/         (@modelcontextprotocol/sdk, transformers, zod, typescript, etc.)
     ├─ rebuild-pty.js                     (native addon rebuild for current platform)
     └─ npm run build in mcp-server/       (tsc compiles src/ → dist/)

synabun
  └─ setup.js runs:
     ├─ checks Node.js >= 22
     ├─ deps already installed ✓ (skip)
     ├─ Playwright Chromium check (installs on first run if missing)
     ├─ MCP already built ✓ (skip)
     ├─ starts Neural Interface server on :3344
     └─ opens browser → onboarding wizard (first run) or main page
```
